PROMPT DE DESPLIEGUE DE WEB-APP EN VERCEL

Usa este bloque como referencia operativa para desplegar el frontend del proyecto Gym Tracker en Vercel sin dominio propio de pago.

OBJETIVO

Publicar el frontend `web-app` gratis en Vercel usando la URL automatica `.vercel.app`, dejando la app accesible online aunque todavia no exista dominio personalizado.

CARPETA CORRECTA A DESPLEGAR

La carpeta que se despliega en Vercel es:

`C:\claude\CARRERA PROFESIONAL\gym-tracker-system\web-app`

No se despliega la raiz completa del monorepo. En Vercel hay que indicar como Root Directory:

`web-app`

CONFIGURACION RECOMENDADA EN VERCEL

- Framework Preset: Next.js
- Root Directory: `web-app`
- Build Command:
  `npm run build`
- Install Command:
  `npm install`
- Output Directory:
  dejar el valor por defecto de Next.js

VARIABLES DE ENTORNO NECESARIAS

En Vercel, dentro del proyecto del frontend, configurar:

- `NEXT_PUBLIC_APP_NAME=Gym Tracker`
- `NEXT_PUBLIC_API_URL=https://tu-api-publica.com`

Si la API todavia no esta publicada, para pruebas iniciales se puede dejar:

- `NEXT_PUBLIC_API_URL=http://localhost:8000`

pero eso no sirve como solucion final en produccion.

FLUJO RECOMENDADO

1. Subir el repositorio a GitHub
2. Entrar en Vercel
3. Importar el repositorio
4. Seleccionar el proyecto
5. En Root Directory elegir `web-app`
6. Revisar variables de entorno
7. Lanzar el deploy
8. Obtener la URL gratuita `.vercel.app`

EJEMPLO DE URL GRATUITA

Vercel asignara una URL parecida a:

- `gym-tracker-system.vercel.app`
- `gym-tracker-luis.vercel.app`

La URL exacta la decide Vercel segun el nombre del proyecto y disponibilidad.

CUANDO USAR DOMINIO PROPIO

No hace falta comprar dominio ahora. Para MVP o pruebas privadas, la opcion correcta es usar la URL gratuita de Vercel.

Solo cuando la app ya este lista o presentable, valorar:

- comprar un dominio
- conectar `app.midominio.com`
- o conectar `gym.midominio.com`

VENTAJAS DE EMPEZAR CON VERCEL GRATIS

- no pagas dominio al principio
- despliegue muy rapido
- HTTPS automatico
- URL publica inmediata
- ideal para validar el MVP

LIMITACION ACTUAL

Aunque el frontend ya compila correctamente, el producto completo aun no estara plenamente operativo en internet hasta que:

- la API tambien se publique
- `NEXT_PUBLIC_API_URL` apunte a esa API publica
- se conecten login, registro y dashboard a datos reales

ULTIMO PROBLEMA DETECTADO

El usuario necesitaba una aclaracion practica sobre como publicar la web sin comprar dominio y que significaba `app.tudominio.com`, porque eso podia sonar abstracto y poco util para arrancar el MVP.

COMO SE HA CORREGIDO

- Se ha definido la carpeta exacta que debe desplegarse en Vercel: `web-app`
- Se ha aclarado que no hace falta dominio propio para empezar
- Se ha fijado como estrategia recomendada usar la URL gratuita `.vercel.app`
- Se han dejado indicadas las variables de entorno minimas del frontend
